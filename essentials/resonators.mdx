## Temporal Resonator Overview

Temporal resonators are specialized devices that generate precisely tuned electromagnetic fields capable of inducing resonance in temporal field structures. They are the primary tool for all TRE operations.

<Info>
  The first functional temporal resonator (TR-1000) was developed in 1996 by the Heisenberg Institute. Modern resonators are 1000× more powerful and precise.
</Info>

### Array Setup

```python
from tre_sdk import ResonatorArray

# Define array geometry
array = ResonatorArray([
    '/dev/tres0',
    '/dev/tres1',
    '/dev/tres2'
])

# Configure triangular geometry
array.set_geometry(
    type='triangular',
    spacing=2.0,  # meters
    orientation=(0, 0, 0)  # euler angles
)

# Synchronize phase across array
array.synchronize_phase(
    reference='/dev/tres0',
    tolerance=0.1  # degrees
)

# Test array coherence
coherence = array.test_coherence()
print(f"Array coherence: {coherence:.4f}")
```

## Troubleshooting

<Tabs>
  <Tab title="Frequency Lock Issues">
    **Symptoms**: Cannot achieve frequency lock

    **Diagnostics**:
    ```bash
    tre-cli diagnostics frequency --device /dev/tres0
    ```

    **Common Causes**:
    - PLL bandwidth too narrow
    - Reference oscillator drift
    - Temperature instability
    - External interference

    **Solutions**:
    ```bash
    # Increase PLL bandwidth
    tre-cli config set pll.bandwidth 0.1

    # Recalibrate reference
    tre-cli calibration reference --auto

    # Check temperature
    tre-cli sensors temperature
    # Should be 23±1°C
    ```
  </Tab>

  <Tab title="Power Output Issues">
    **Symptoms**: Insufficient or unstable power output

    **Diagnostics**:
    ```bash
    tre-cli diagnostics power --sweep --device /dev/tres0
    ```

    **Common Causes**:
    - Amplifier saturation
    - Cooling system inadequate
    - Power supply voltage droop
    - Cable/connector issues

    **Solutions**:
    ```bash
    # Check amplifier status
    tre-cli diagnostics amplifier

    # Verify cooling
    tre-cli cooling verify --flow-rate --temperature

    # Test power supply
    tre-cli diagnostics power-supply

    # Inspect connections
    tre-cli diagnostics connections
    ```
  </Tab>

  <Tab title="Communication Errors">
    **Symptoms**: Device not responding or intermittent connectivity

    **Diagnostics**:
    ```bash
    tre-cli diagnostics communication --device /dev/tres0
    ```

    **Solutions**:
    ```bash
    # Reset USB connection
    tre-cli device reset --interface usb

    # Reload drivers
    sudo modprobe -r tre_usb
    sudo modprobe tre_usb

    # Check firmware version
    tre-cli firmware version
    # Update if needed
    tre-cli firmware update --latest
    ```
  </Tab>
</Tabs>
